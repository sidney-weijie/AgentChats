# AgentChat · 监控审计与场景化启动（08）

本文档补充两个能力域：
**A. 可观测性与审计**——结构化日志、指标、链路追踪、审计闭环；
**B. 场景化启动（Launch Scenario）**——外部系统跳转进入、预置工作区与数据、受限探索、
支持重置与复制副本。

---

## A. 可观测性与审计

### A.1 分层模型

| 层 | 内容 | 载体 | 保留策略 |
|---|---|---|---|
| 审计（合规） | 谁在何时对什么做了什么 | `audit_logs` 表（只增不改） | ≥180 天，可按需归档到对象存储 |
| 应用日志 | 结构化 JSON 日志，含关联 ID | stdout → 采集（Loki/ELK 均可） | 30 天 |
| 指标 | 计数/时延/额度水位 | Prometheus `/metrics` | 按监控平台策略 |
| 链路追踪 | 一次 run 内部各阶段耗时 | OpenTelemetry | 采样保留 |

### A.2 关联 ID 体系（所有层共用，排障的关键）

```
request_id  每个 HTTP 请求生成（中间件），响应头回传 X-Request-Id
run_id      一次生成过程；SSE 事件、日志、审计、追踪全部携带
thread_id / user_id / iss(接入方)   业务维度
```

实现：FastAPI 中间件生成 `request_id` 放入 `contextvars`；日志用 
`structlog`（或 logging + JSON formatter）自动注入上下文字段：

```python
log.info("tool_call", tool="Bash", run_id=run.id, thread_id=..., user_id=...,
         input_digest=sha256_prefix(tool_input), duration_ms=132)
```

规范：**日志里不落敏感原文**——工具输入/输出只记摘要与截断样本（前 200 字符），
完整内容进 `audit_logs.detail`（该表访问本身受管理端权限与审计保护）；
API key/凭据一律打码。

### A.3 审计事件目录（audit_logs.event 枚举）

```
tool_call / tool_denied / tool_error      每次工具调用及其结果
approval_requested / approval_decided     人工审批全流程（含 remember 标记）
hook_triggered                            Hook 拦截/放行（策略 key + 命中规则）
run_started / run_finished / run_stopped  含 usage 与终态
login / login_failed / token_refresh      账户事件
config_change                             权限档案/Hook/命令/全局配置的变更（before/after 快照）
scenario_launch / scenario_reset / scenario_duplicate   场景生命周期（见 B 部分）
file_preview / file_download              工作区文件访问（含 path）
share_created / share_revoked
```

写入位置全部收敛在服务层（runner.dispatch、approvals、permissions、admin API），
禁止散落在路由层随手写——保证事件目录可枚举、可校验。

### A.4 指标（Prometheus 命名）

```
agentchat_runs_total{status}                 run 计数（completed/stopped/error/timeout）
agentchat_run_duration_seconds{quantile}     run 时长分布
agentchat_tool_calls_total{tool,decision}    工具调用（decision=allowed/denied/asked）
agentchat_approval_wait_seconds              审批等待时长（超时率的来源）
agentchat_tokens_total{direction}            input/output token 累计
agentchat_sse_active_connections             当前 SSE 连接数
agentchat_quota_exhausted_total              额度拒绝计数（429）
agentchat_scenario_launch_total{template,iss}
```

告警建议：error 率 >2%（5min）、审批超时率 >20%、单用户 tokens 突增 5×日常、
SSE 连接数逼近实例上限。

### A.5 链路追踪

社区已有现成的 SDK 插桩包 **`openinference-instrumentation-claude-agent-sdk`**
（对 `query()`/`ClaudeSDKClient` 生成 OTel AGENT span，经 hook 注入生成工具子 span，
含 token 计数），OTLP 导出到任意 collector。接入成本低，建议 M7 一并做：

```python
from openinference.instrumentation.claude_agent_sdk import ClaudeAgentSDKInstrumentor
ClaudeAgentSDKInstrumentor().instrument()   # 需配 OTEL_EXPORTER_OTLP_ENDPOINT
```

自有代码只需再给"审批等待"和"物化工作区"两个阶段手动加 span，
一条 trace 即可看清一次 run 的完整耗时构成。

### A.6 管理端呈现

管理控制台底部的"审计日志"视图（前端已实现演示）：按用户/事件类型/时间过滤，
逐行展示 时间/用户/事件/摘要/run_id；点击 run_id 可跳转该 run 的完整事件序列
（`GET /admin/runs/{id}/audit`）。config_change 类事件展示 before/after diff。

## B. 场景化启动（Launch Scenario）

### B.1 场景定义

外部系统（BI 平台、工单系统、数据门户……）中用户点击"用 AI 分析"，跳转进入
AgentChat 时：**用户身份已就绪、工作区已预置好数据文件、能力已按场景收窄**，
用户直接开始探索。支持**重置**（回到刚跳转进来的初始状态）与**复制副本**
（新会话 + 同一份初始数据）。

这是 07 文档嵌入体系的上层应用：身份复用 embed_token 机制，新增"模板 + 种子"两个概念。

### B.2 数据模型（新增两张表 + threads 扩展）

```sql
launch_templates(                        -- 场景模板（管理员预先配置）
  id VARCHAR(36) PK,
  embed_client_id FK,                    -- 归属哪个接入方
  name VARCHAR(100),                     -- "数据分析场景"
  system_prompt TEXT,                    -- 场景系统提示（分析师人设/输出规范）
  greeting_template TEXT,                -- 首条欢迎消息模板, 支持 {{files}} 等占位符
  model VARCHAR(60),                     -- 锁定模型
  scenario_grants JSON,                  -- 场景能力集(见 B.4, 只能收窄不能放大)
  data_spec JSON,                        -- 数据来源声明(见 B.3)
  max_reset_count INT DEFAULT 20,        -- 防滥用
  enabled BOOLEAN, created_at, updated_at
)

workspace_seeds(                         -- 种子快照：重置/复制的恢复依据
  id VARCHAR(36) PK,
  thread_id FK,                          -- 首个会话; 副本会话复用同一 seed
  template_id FK,
  manifest JSON,                         -- [{path, sha256, size, storage_key}]
  storage_prefix VARCHAR(300),           -- 种子文件在对象存储/本地种子区的位置
  created_at
)

threads 增列:
  template_id VARCHAR(36) NULL,          -- 非空即场景会话
  seed_id VARCHAR(36) NULL,
  reset_count INT DEFAULT 0
```

### B.3 启动流程

```
外部系统                                AgentChat
  │ 签发 launch_token = embed_token 扩展:  │
  │   + template_id                       │
  │   + params: {title, 上下文变量}        │
  │   + data: [{name, url, sha256}] 或     │
  │           {push: true}(随后主动推送)    │
  │──浏览器跳转 /launch?token=... ────────►│ POST /api/v1/launch
  │                                       │ 1 验签(同 embed, 加 template 归属校验)
  │                                       │ 2 upsert 联合用户
  │                                       │ 3 建 thread(template_id) + 独立 workspace
  │                                       │ 4 拉取 data.url 文件 → 校验 sha256/大小/类型
  │                                       │   → 落 workspace/uploads/ 同时存一份到种子区
  │                                       │ 5 写 workspace_seeds(manifest)
  │                                       │ 6 渲染 greeting 为首条 assistant 消息
  │                                       │ 7 audit: scenario_launch
  │                                       │◄ 302 → /chat/{thread_id} (带正式 token)
```

数据入区的两种方式：**拉**（launch_token 里带预签名 URL，AgentChat 主动下载，适合
对象存储）或**推**（先 `POST /launch` 拿到 thread_id，宿主再调
`POST /threads/{id}/seed-files` multipart 上传，全部就绪后调 `POST /threads/{id}/seal`
封存种子）。两种都必须过：大小上限、类型白名单、sha256 校验、病毒扫描钩子（预留）。

### B.4 场景下的能力限制（"限制用户的发挥"）

场景能力集 `scenario_grants` 进入权限合并链，**位置在管理员用户档案之后、
且只做交集（只能收窄，永远不能放大管理员档案）**：

```
① 系统锁定规则
② 管理员用户档案 (user_permission_profiles)
③ 场景模板 scenario_grants        ← 新增: effective = ② ∩ ③
④ project 规则
⑤ user 规则
```

典型的数据分析场景配置：

```json
{
  "tools": {"bash": true, "file_write": true, "web_search": false},
  "bash_profile": "data_analysis",      // 映射一组 Hook: 只放行 python/pandas 等白名单命令
  "model_locked": true,                  // UI 隐藏模型选择器
  "mode": "default",                     // 强制权限模式
  "slash_commands": ["compact", "usage"],// 命令面板收窄
  "hide_settings": true,                 // 场景会话不展示"我的设置"入口
  "network": "none"                      // 沙箱断网(数据不出域)
}
```

实现要点：`permissions.compile(ctx)` 里当 `thread.template_id` 非空时加载模板并做
交集；**UI 的隐藏只是体验，强制永远在服务端**（与既有原则一致）；
`network: none` 落在沙箱层（04 文档容器网络策略），这是"数据分析场景数据不出域"
的真正保证。

### B.5 重置与复制副本

```
POST /threads/{id}/reset
  校验: 是场景会话 + reset_count < max_reset_count + 无 running run(有则先 409 或级联 stop)
  动作: 1 结束/废弃当前 branch(不 resume, transcript 保留供审计)
        2 清空 workspace(保留目录) → 按 seed manifest 从种子区恢复文件 → 逐一校验 sha256
        3 重新物化 .claude/ 配置; 重新渲染 greeting 为新的首条消息
        4 reset_count += 1; audit: scenario_reset
  语义: 与"重新从外部系统跳转进来"完全一致, 但不需要宿主重新签发 token

POST /threads/{id}/duplicate
  动作: 新建 thread(同 template_id, 同 seed_id) + 新 workspace ← 同一 seed 物化
        标题加"（副本）"; audit: scenario_duplicate
  语义: 原会话的探索过程完整保留, 新会话从同一份初始数据重新开始
  说明: 副本共享 seed(只读), 不共享任何对话或工作区改动
```

种子区文件只读、带引用计数：所有引用它的 threads 删除后才可清理（后台任务）。

### B.6 API 汇总（并入 05 文档）

```
POST /launch                              launch_token → 302 /chat/{thread_id}
POST /threads/{id}/seed-files             宿主推送种子数据(封存前)
POST /threads/{id}/seal                   封存种子, 会话可用
POST /threads/{id}/reset                  恢复初始状态
POST /threads/{id}/duplicate → {thread_id} 复制副本
GET/POST/PATCH/DELETE /admin/launch-templates   模板管理(管理端)
```

### B.7 前端行为（已在 AgentChatUI.jsx 实现演示）

- 场景会话头部：蓝色徽标"数据分析场景 · 来自 BI 报表平台" + **重置** / **复制副本** 按钮
  （重置有确认弹窗，生成中先停止再重置）；
- 输入区：模型选择器替换为"🔒 已锁定"标识，联网开关隐藏；
- 首条消息为渲染后的 greeting（列出已导入的数据文件）；
- 文件树中种子数据在 `uploads/` 下可预览/下载（受 workspace_view 授权约束）。

### B.8 排期

依赖 07 的 embed 基建（M-Embed）。其后插入 **M-Scenario（~5d）**：
模板表/种子表 + /launch 全流程 + reset/duplicate + scenario_grants 交集编译 + 审计事件。
验收：从一个 demo 宿主页带数据跳转 → 分析对话 → 修改工作区文件 → 重置后文件哈希
与首次进入一致 → 复制副本后两个会话互不影响；未授权模板/超次数重置被拒并有审计记录。
可观测性（A 部分）不单独设里程碑：A.2/A.3 从 M1 起随代码落地，指标与追踪并入 M7。
