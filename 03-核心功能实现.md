# AgentChat · 核心功能实现

> 涉及 Agent SDK 具体参数名的地方以你安装版本的官方文档为准（SDK 迭代快），

## 0. Agent Runner：所有功能的公共底座

```python
# runner.py — 每次生成的统一入口
async def execute_run(run: Run, prompt: str | list, ctx: RunContext):
    thread, branch = ctx.thread, ctx.branch
    materialize_workspace(thread, ctx.db)      # 物化命令/subagent/MCP 配置 (见§4)

    options = ClaudeAgentOptions(
        cwd=thread.workspace_path,
        resume=branch.agent_session_id or None,
        fork_session=ctx.fork,                  # 编辑分支时 True
        model=ctx.model,
        allowed_tools=PermissionService.effective_tools(ctx),
        disallowed_tools=PermissionService.deny_list(ctx),
        permission_mode=ctx.permission_mode,
        hooks=HookService.build(ctx),           # 见 04 文档
        can_use_tool=ApprovalService.callback(ctx),
        max_turns=ctx.max_turns,
    )
    try:
        async for msg in query(prompt=prompt, options=options):
            await dispatch(msg, ctx)            # 落库 + 写入 SSE 事件缓冲
    except asyncio.CancelledError:
        finalize(run, "stopped"); raise
    except Exception as e:
        finalize(run, "error", str(e)); raise
    else:
        finalize(run, "completed")
```

`dispatch` 按消息类型处理（对应 01 文档 §3 的 SSE 事件表）：
`init`→写回 session_id；文本/思考增量→缓冲 200ms 批量落库、实时写 run 事件缓冲(SSE 消费)；
`tool_use`/`tool_result`→审计 + 推卡片事件；`result`→结算 usage 到 runs 与 usage_daily。

## 1. 对话核心

### 1.1 发送 + 流式
`POST /threads/{id}/messages` → 校验（鉴权→当日额度→branch 无 running run→拿 branch 锁）
→ 落 user message + run → `asyncio.create_task(execute_run(...))` 注册进 `running_tasks[run_id]`
→ 立即返回 `{run_id}`，前端随即 `GET /runs/{run_id}/stream` 订阅 SSE。

### 1.2 停止
`POST /runs/{id}/stop` → `running_tasks[run_id].cancel()`。
CancelledError 在 `query()` 的下一个 await 点抛出；已生成部分保留为一条"未完成"的 assistant 消息。
（多实例部署后改为 Redis pub/sub 通知持有实例执行 cancel。）

### 1.3 编辑消息 → 分支 / 重新生成
- 编辑第 N 条用户消息：新建 branch（`parent_branch_id` = 当前分支，
  `forked_at_message_id` = 该消息），调用时 `resume=旧session_id, fork_session=True,
  resume_session_at=前一条assistant消息的agent_message_uuid`，拿到新 session_id 写回新分支。
- 重新生成：同上，但 fork 点定位到用户消息本身、prompt 复用原文。
- 版本切换：`GET /threads/{id}/branches` 按 fork 点分组，前端 `< 1/2 >` 切换 `is_head`。

### 1.4 会话管理
新建（首条消息才建 branch，避免空会话）、列表（`pinned DESC, last_active_at DESC` 游标分页，
时间分组由前端算）、重命名（首轮结束后用 Haiku 自动起标题）、软删除/归档、
搜索（SearchBackend 接口：SQLite fts5 → MySQL 换实现）。

## 2. 生成参数

- **模型切换**：session 只是历史，模型是逐次调用参数——同会话内自由切换，随请求传 `model`。
- **深度思考**：请求带 `thinking: true/预算档位` → 映射 SDK 思考配置；
  thinking 内容单独作为 content 分段，前端折叠。
- **联网搜索/工具开关**：映射为本轮 `allowed_tools` 的增减，同样不需要开分支。
- **用量/限流**：调用**前**查 `usage_daily` 校验额度，超限 429；`result` 后 UPSERT 结算。

## 3. 斜杠命令

三类命令，两条处理路径：

**A. 本地动作类**（不进 Agent）：`/model` `/permissions` `/cost` `/help` 打开对应面板；
`/clear` = 新开 branch 且不 resume（保留旧历史可回看）。前端命令表里标记 `local: true`。

**B. 透传类**：`/compact`、自定义 Skill、MCP prompt——原样把 `"/xxx args"` 作为 prompt
交给 `query()`，SDK 自己解析。后端唯一职责：保证 cwd 下存在对应定义文件。

命令来源聚合接口 `GET /threads/{id}/commands`：内置表(硬编码) + `slash_commands` 表
+ 最近一次 init 消息里 SDK 上报的 `slash_commands` 列表（含 MCP prompts）取并集去重。

**自定义命令存储与物化**（DB 为真相源，文件为运行前产物）：

```python
def materialize_commands(thread, db):
    base = Path(thread.workspace_path) / ".claude" / "skills"
    for cmd in enabled_commands_for(thread, db):   # user/project/global 三级合并
        d = base / cmd.name; d.mkdir(parents=True, exist_ok=True)
        (d / "SKILL.md").write_text(yaml_frontmatter(
            description=cmd.description, argument_hint=cmd.argument_hint,
            allowed_tools=cmd.allowed_tools, model=cmd.model) + cmd.body)
    # 用 (id, updated_at) 哈希做缓存, 未变更则跳过写盘
```

前端交互：输入 `/` 弹命令面板（模糊搜索，来源徽标 builtin/skill/mcp），
选中后若有 `argument_hint` 则等待补参，回车按 local 标记分流。

## 4. Subagent / MCP / Plan Mode

- **Subagents**：`subagents` 表 CRUD；运行前映射到 SDK 的 agents 配置项。
  消息流里 `SubagentStart/Stop` 事件渲染成"已委派 → xxx"的嵌套卡片。
- **MCP**：`mcp_servers` 表 CRUD（env 加密存储）；映射 `mcp_servers` 选项。
  设置页提供"测试连接"按钮（起一次最小 query 看 init 里是否出现该 server 的工具）。
  其 tools/prompts 自动进工具面板与命令面板。
- **Plan Mode**：会话级开关 → `permission_mode="plan"`。UI 上以"计划卡片 + 确认执行"
  呈现：确认后同一会话再发一轮、切回正常权限模式执行。

## 5. 文件与 Artifacts

- **上传**：文件落 `workspace_path/uploads/`（Agent 经 Read/Glob 天然可见）+ `attachments` 记录；
  图片/PDF 也可直接作为多模态输入拼进 prompt。
- **Agent 产物下载**：run 结束后 diff 工作区新增文件 → 登记 `attachments(direction=output)`
  → 消息流尾部渲染下载卡片。
- **文件树侧栏**：`GET /threads/{id}/files` 列 workspace 目录（限深度/数量），
  这是把"Claude Code 在操作哪个目录"可视化的关键组件。
- **Artifacts 分期**：一期解析特定标记渲染只读侧栏（md/html/svg/code）；
  二期 iframe sandbox 跑可交互 HTML/React；三期版本链 + 分享。

## 6. Projects / Memory / 分享

- Projects：`instructions` 进系统提示；项目文件同步进 workspace（或 additional dirs）。
- Memory：会话结束异步摘要 → `memory_entries`；新会话取 active 条目注入系统提示；
  设置页可查看/编辑/删除（不可做成黑盒）。
- 分享：生成 `share_token` + **消息快照**（不实时联动原会话），公开只读页，可撤销。

## 7. Claude Code 能力覆盖优先级

- **P0**：文件工具 + Bash（配权限）/ 流式 / resume / 分支 / `/clear /compact /model /cost /help` / 文件树
- **P1**：自定义命令 / Hooks / 细粒度权限 + 交互审批 / Subagents / MCP 管理 / Plan Mode
- **P2**：插件市场、文件 checkpoint 回滚、后台任务与并行 worktree、定时 Managed Agents

P2 属重度工程师工作流，建议观察真实需求再投入。
