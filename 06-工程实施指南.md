### 1.1 SDK 关键语义（必读，直接影响实现正确性）

以下来自 0.2.110 版官方 README/文档，与直觉不同、容易写错：

1. **`allowed_tools` 是"自动放行白名单"，不是工具裁剪**。列入 `allowed_tools` 的工具
   跳过询问直接执行；未列入的工具**依然存在**，会落入 `permission_mode` 和
   `can_use_tool` 的判定链。**要真正禁掉一个工具，必须放进 `disallowed_tools`**。
   → 我们的 `PermissionService` 编译规则时：管理员未授予的工具（如 bash=false）
   必须写入 `disallowed_tools`，而不是简单地"不加入 allowed_tools"。
2. **hooks 与自定义（in-process MCP）工具需要 `ClaudeSDKClient`**，简单的 `query()`
   函数不支持这两项。本项目 Hook 是核心功能，所以 Runner 统一用 `ClaudeSDKClient`。
3. **`can_use_tool` 回调**在权限链末端兜底（Hook → deny → allow → ask → mode → canUseTool），
   我们的交互式审批挂在这里。
4. session transcript 与 `cwd` 绑定，`resume` 时必须传**同一个** `cwd`。
5. Python 包体较大（~75MB，含捆绑 CLI），构建 Docker 镜像时留意层缓存。

### 3.2 events.py — run 事件缓冲与订阅（SSE 的数据源）

```python
import asyncio
from dataclasses import dataclass, field

@dataclass
class RunChannel:
    events: list = field(default_factory=list)      # [(seq, event_name, json_str)]
    cond: asyncio.Condition = field(default_factory=asyncio.Condition)
    finished: bool = False

CHANNELS: dict[str, RunChannel] = {}                # run_id -> channel（单实例内存态）

async def emit(run_id: str, name: str, data: dict):
    ch = CHANNELS.setdefault(run_id, RunChannel())
    async with ch.cond:
        ch.events.append((len(ch.events) + 1, name, json.dumps(data, ensure_ascii=False)))
        if name == "run_status" and data["data"].get("status") != "running":
            ch.finished = True
        ch.cond.notify_all()

async def subscribe(run_id: str, after_seq: int = 0):
    """异步生成器：从 after_seq 之后开始产出事件，直到 run 结束。"""
    ch = CHANNELS.setdefault(run_id, RunChannel())
    idx = after_seq
    while True:
        async with ch.cond:
            while len(ch.events) <= idx and not ch.finished:
                await ch.cond.wait()
            batch = ch.events[idx:]; idx = len(ch.events)
            done = ch.finished and idx >= len(ch.events)
        for seq, name, payload in batch:
            yield seq, name, payload
        if done:
            return
```

> 多实例部署时，把 `emit` 改为投递 Redis Stream、`subscribe` 改为 XREAD——
> 接口不变，只换实现（对应 04 文档"存储面扩展点"）。

### 3.3 runs.py — SSE 端点

```python
from sse_starlette.sse import EventSourceResponse

@router.get("/runs/{run_id}/stream")
async def stream_run(run_id: str, request: Request, user=Depends(current_user)):
    run = await get_run_checked(run_id, user)        # 校验归属, 404/403
    last_id = int(request.headers.get("last-event-id", 0) or 0)

    async def gen():
        # 心跳与事件二选一推进；EventSourceResponse 自带 ping 支持
        async for seq, name, payload in events.subscribe(run_id, after_seq=last_id):
            if await request.is_disconnected():
                return                               # 只停推送，不取消 run
            yield {"id": str(seq), "event": name, "data": payload}

    return EventSourceResponse(gen(), ping=15,
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})
```

### 3.4 runner.py — Agent Runner（核心，ClaudeSDKClient 版）

```python
from claude_agent_sdk import ClaudeSDKClient, ClaudeAgentOptions

RUNNING_TASKS: dict[str, asyncio.Task] = {}          # run_id -> task

async def start_run(run, prompt, ctx):
    task = asyncio.create_task(_execute(run, prompt, ctx))
    RUNNING_TASKS[run.id] = task
    task.add_done_callback(lambda _: RUNNING_TASKS.pop(run.id, None))

async def _execute(run, prompt, ctx):
    workspace.materialize(ctx.thread, ctx.db)        # 物化命令/subagent 配置
    perm = permissions.compile(ctx)                  # 见 3.5
    options = ClaudeAgentOptions(
        cwd=ctx.thread.workspace_path,
        resume=ctx.branch.agent_session_id or None,
        fork_session=ctx.fork,
        model=ctx.model,
        allowed_tools=perm.allowed,
        disallowed_tools=perm.disallowed,            # 关键: 未授权工具必须在这里
        permission_mode=perm.mode,
        hooks=hooks.build(ctx),
        can_use_tool=approvals.make_callback(run, ctx),
        max_turns=ctx.max_turns,
    )
    status, err = "completed", None
    try:
        async with ClaudeSDKClient(options=options) as client:
            await client.query(prompt)
            async for msg in client.receive_response():
                await dispatch(msg, run, ctx)        # 见 3.4.1
    except asyncio.CancelledError:
        status = "stopped"; raise
    except Exception as e:
        status, err = "error", str(e)
    finally:
        await finalize(run, status, err, ctx)        # 落库 + emit run_status 收尾事件

async def stop_run(run_id: str):
    """POST /runs/{id}/stop 的实现。多实例时改为 Redis pub/sub 通知持有实例。"""
    task = RUNNING_TASKS.get(run_id)
    if task: task.cancel()
```

#### 3.4.1 dispatch：SDK 消息 → 落库 + SSE 事件

按 `type(msg)` 分派（SDK 消息类型名以 0.2.110 的 `claude_agent_sdk` 导出为准，
如 `SystemMessage/AssistantMessage/ResultMessage` 与内容块 `TextBlock/ToolUseBlock/
ToolResultBlock/ThinkingBlock`——开工第一天先写一个"打印所有消息类型"的探针脚本
跑一轮真实对话，把确切的类型与字段固化成常量表再写 dispatch）：

| SDK 消息 | 动作 |
|---|---|
| system init | `branch.agent_session_id ??= msg.session_id`；记录 init 上报的可用工具/命令列表 |
| assistant 文本块 | 追加进内存缓冲，emit `text_delta`；每 200ms 或每 N 字符批量 UPDATE messages |
| thinking 块 | 同上，事件名 `thinking_delta` |
| tool_use 块 | emit `tool_use` + 写 audit_logs |
| tool_result 块 | emit `tool_result`（对超长输出截断到 8KB 再下发，全文只入审计） |
| result | 结算 usage：写 messages.usage / runs / usage_daily；**按 show_cost 裁剪后**再 emit `run_status` |

### 3.6 approvals.py — 交互式审批（canUseTool → SSE → REST 决议）

```python
PENDING: dict[str, asyncio.Future] = {}

def make_callback(run, ctx):
    async def can_use_tool(tool_name, tool_input, context):
        ap = await create_pending(run, tool_name, tool_input)
        fut = asyncio.get_running_loop().create_future()
        PENDING[ap.id] = fut
        await events.emit(run.id, "approval_request", envelope(run, {
            "approval_id": ap.id, "tool_name": tool_name, "tool_input": tool_input}))
        try:
            decision, remember = await asyncio.wait_for(fut, timeout=300)
        except asyncio.TimeoutError:
            await mark(ap, "timeout")
            return deny_result("审批超时自动拒绝")
        finally:
            PENDING.pop(ap.id, None)
        if remember and decision == "allow":
            await permissions.remember(ctx.user, tool_name, tool_input)
        return allow_result() if decision == "allow" else deny_result("用户拒绝")
    return can_use_tool
# POST /approvals/{id}/decide → PENDING[id].set_result((decision, remember))
# 返回类型用 SDK 的 PermissionResultAllow / PermissionResultDeny
```

### 3.7 workspace.py — 目录安全与文件树（workspace_view 门禁）

```python
def safe_resolve(thread, rel: str) -> Path:
    base = Path(thread.workspace_path).resolve()
    target = (base / rel.lstrip("/")).resolve()      # resolve 同时消解 .. 与符号链接
    if not target.is_relative_to(base):
        raise HTTPException(400, "路径越界")
    return target

@router.get("/threads/{tid}/files")
async def list_files(tid: str, path: str = "", user=Depends(current_user)):
    thread = await get_thread_checked(tid, user)
    profile = await admin_profile(user)
    if not profile.tool_grants.get("workspace_view"):
        raise HTTPException(403, "未授权查看工作区")   # 前端同时按 /me.features 隐藏入口
    root = safe_resolve(thread, path)
    items = []
    for p in sorted(root.iterdir())[:200]:           # 限量; 子目录懒加载
        items.append({"name": p.name, "type": "dir" if p.is_dir() else "file",
                      "size": p.stat().st_size if p.is_file() else None})
    return {"path": path, "items": items}
```

### 3.8 hooks.py — 内置策略示例

```python
BUILTIN_POLICIES = {}
def policy(key):
    def reg(fn): BUILTIN_POLICIES[key] = fn; return fn
    return reg

@policy("deny_dangerous_bash")
def deny_dangerous_bash(params):
    patterns = [re.compile(p) for p in params.get("patterns", DEFAULT_BLACKLIST)]
    async def hook(input_data, tool_use_id, context):
        cmd = input_data.get("tool_input", {}).get("command", "")
        if any(p.search(cmd) for p in patterns):
            return {"hookSpecificOutput": {
                "hookEventName": "PreToolUse",
                "permissionDecision": "deny",
                "permissionDecisionReason": "命中危险命令拦截规则"}}
        return {}
    return hook
```

## 4. 里程碑与验收标准

| 里程碑 | 内容 | 验收标准 |
|---|---|---|
| **M0 骨架**（~3d） | 项目结构/配置/DB/Alembic/JWT 注册登录；SDK 探针脚本 | `alembic upgrade head` 建表成功；探针脚本能跑通一轮真实 SDK 对话并打印全部消息类型 |

| **M4 审批与 Hook**（~4d） | canUseTool 审批链 + 内置策略 + audit_log 强制 | 写文件触发审批卡片；允许/拒绝/超时三条路径都正确；"记住"生成规则下次不再问；危险命令被 Hook 拦截 |
| **M5 命令与工作区**（~4d） | 斜杠命令 CRUD/物化/聚合 + 文件上传 + 文件树(含 workspace_view 门禁) | 网页建自定义命令→对话里 `/xxx` 生效；未授权用户 files 接口 403 且无入口；路径穿越用例（`../`、符号链接）全部被拒 |

| **M7 收尾**（~3d） | 用量限额/管理端报表/部署脚本/压测 | 超限返回 429；nginx 配置下 SSE 不被缓冲；20 并发用户流式无串流错乱 |

## 6. 常见坑清单

1. `allowed_tools` 不禁用工具——见 §1.1，权限测试必须覆盖。
2. SQLite 写并发：所有写操作走单 Session 短事务；后台任务与请求竞争写时靠
   busy_timeout 兜底；压测阶段若频繁 `database is locked`，把落库改为单写队列。
3. SSE 被缓冲：本地 uvicorn 直连正常、上了 nginx 就"不流"，九成是没关 proxy_buffering。
4. EventSource 与鉴权：原生 EventSource 不能带 Authorization 头——统一用 fetch 流
   （前端参考实现已内置在 AgentChatUI.jsx 顶部注释）。
5. 忘记绑定 cwd：resume 时 cwd 与首轮不一致会找不到 session。
6. 审批 Future 泄漏：超时/异常路径必须 pop PENDING，否则内存缓慢增长。
7. 工具输出直发前端：超长 tool_result 要截断，完整内容只进审计表。
8. 前端信任：features 只是渲染开关，任何强制都在服务端 compile 里完成。
9. Docker 镜像：claude-agent-sdk 的 wheel ~75MB（含捆绑 CLI），
   把 `pip install` 放在独立层并善用缓存；确认运行用户对捆绑 CLI 有执行权限。
10. 计费合规：对外提供服务必须走 API key，不能让终端用户以 claude.ai 订阅身份使用。
