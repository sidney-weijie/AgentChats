# AgentChat · 数据模型设计

技术基线：SQLAlchemy 2.0 + Alembic。第一期 SQLite，上线迁 MySQL 8.x。

## 1. SQLite / MySQL 双兼容规范（所有表必须遵守）

| 规范 | 做法 |
|---|---|
| 主键 | 应用层生成 UUID，`String(36)`；禁止依赖数据库自增 |
| JSON | `sqlalchemy.JSON`；禁止 jsonb / 原生 JSON 函数写查询 |
| 枚举 | `String` + 应用层常量；禁止数据库 ENUM |
| 布尔 | `Boolean`（SQLAlchemy 抹平 0/1 与 TINYINT 差异） |
| 时间 | `DateTime`，统一 UTC，应用层带 tz 转换 |
| 外键 | 显式 `ForeignKey`；SQLite 连接后执行 `PRAGMA foreign_keys=ON` |
| 长文本 | `Text`；MySQL 侧对应 `LONGTEXT`（Alembic 方言映射自动处理） |
| 索引名 | 全库统一命名 `ix_{table}_{cols}`，迁移时可对账 |
| Schema变更 | Alembic 一律用 `batch_alter_table()` 写迁移，兼容 SQLite 弱 ALTER |
| 全文检索 | 抽象成 `SearchBackend` 接口：SQLite 用 fts5 虚表实现，MySQL 换 FULLTEXT/外部引擎实现 |

SQLite 初始化 PRAGMA（engine connect 事件里执行）：
`journal_mode=WAL`、`foreign_keys=ON`、`busy_timeout=5000`、`synchronous=NORMAL`。

## 2. 表结构总览（按域分组）

### 2.1 账户域

```sql
users(
  id            VARCHAR(36) PK,
  email         VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255),
  name          VARCHAR(100),
  role          VARCHAR(20)  DEFAULT 'user',     -- user / admin
  plan          VARCHAR(20)  DEFAULT 'free',     -- free / pro / team
  status        VARCHAR(20)  DEFAULT 'active',
  created_at    DATETIME, updated_at DATETIME
)

api_tokens(id PK, user_id FK, token_hash, name, scopes JSON,
           expires_at, last_used_at, created_at)   -- 给自动化/第三方接入用
```

### 2.2 会话域（核心）

```sql
projects(
  id VARCHAR(36) PK, user_id FK, name VARCHAR(200),
  instructions TEXT,                    -- 项目级系统提示
  settings JSON,                        -- 项目级默认: 模型/权限模式/工具集
  created_at, updated_at
)

threads(                                -- 侧边栏里的"一个会话"
  id VARCHAR(36) PK,
  user_id FK, project_id FK NULL,
  title VARCHAR(300),
  pinned BOOLEAN DEFAULT 0,
  archived_at DATETIME NULL,
  deleted_at DATETIME NULL,             -- 软删除
  workspace_path VARCHAR(500),          -- /workspace/{user_id}/{thread_id}/
  created_at, last_active_at,
  INDEX ix_threads_user_active (user_id, deleted_at, last_active_at)
)

branches(                               -- 编辑/重新生成产生的对话分支
  id VARCHAR(36) PK, thread_id FK,
  parent_branch_id FK NULL,
  forked_at_message_id VARCHAR(36) NULL,
  agent_session_id VARCHAR(100),        -- SDK session_id, resume/fork 用
  is_head BOOLEAN DEFAULT 1,            -- 当前展示分支
  model VARCHAR(60),
  created_at,
  INDEX ix_branches_thread (thread_id, is_head)
)

messages(                               -- UI 渲染/搜索/审计用副本
  id VARCHAR(36) PK, branch_id FK,
  role VARCHAR(20),                     -- user / assistant
  content JSON,                         -- 结构化分段, 见 2.5
  agent_message_uuid VARCHAR(100),      -- SDK 消息 uuid → resumeSessionAt 定位
  usage JSON,                           -- {input_tokens,output_tokens,cost_usd}
  created_at,
  INDEX ix_messages_branch (branch_id, created_at)
)

runs(                                   -- 一次"生成", 支撑 stop/审计/结算
  id VARCHAR(36) PK, branch_id FK,
  trigger_message_id FK,
  status VARCHAR(20),                   -- running/completed/stopped/error/timeout
  error TEXT NULL,
  options_snapshot JSON,                -- 本轮实际生效的 model/工具/权限模式快照
  total_cost_usd DECIMAL(10,6),
  started_at, ended_at
)

attachments(id PK, message_id FK NULL, thread_id FK, user_id FK,
            storage_path VARCHAR(500), filename, mime_type, size_bytes,
            direction VARCHAR(10),      -- upload / output(Agent产物)
            created_at)
```

### 2.3 扩展配置域（斜杠命令 / Subagent / MCP）

```sql
slash_commands(
  id PK, owner_type VARCHAR(10),        -- user / project / global
  owner_id VARCHAR(36) NULL,
  name VARCHAR(64),                     -- 不含斜杠
  description VARCHAR(300), argument_hint VARCHAR(100),
  allowed_tools JSON, model VARCHAR(60) NULL,
  body TEXT,                            -- SKILL.md 正文, 支持 $ARGUMENTS
  enabled BOOLEAN DEFAULT 1, created_at, updated_at,
  UNIQUE (owner_type, owner_id, name)
)

subagents(
  id PK, owner_type, owner_id, name VARCHAR(64),
  description VARCHAR(300),
  system_prompt TEXT,
  allowed_tools JSON, model VARCHAR(60) NULL,
  enabled BOOLEAN DEFAULT 1, created_at, updated_at
)

mcp_servers(
  id PK, owner_type, owner_id, name VARCHAR(64),
  transport VARCHAR(20),                -- stdio / sse / http
  config JSON,                          -- {command,args,env} 或 {url,headers}
  enabled BOOLEAN DEFAULT 1,
  status VARCHAR(20) DEFAULT 'unknown', -- 最近一次连通性检查结果
  created_at, updated_at
)
```

> 三张表共同的运行时机制：发起 query 前把配置**物化**到 `workspace_path` 下
> （`.claude/skills/`、subagent/MCP 对应的 options 字段），文件只是产物，DB 是真相源。
> 敏感字段（MCP env 里的 token）用应用层对称加密（Fernet）后入库。

### 2.4 安全域（Hook / 权限 / 审批 / 审计）

```sql
hook_rules(
  id PK, owner_type, owner_id,
  event VARCHAR(40),                    -- PreToolUse/PostToolUse/UserPromptSubmit/Stop...
  matcher VARCHAR(200) NULL,            -- 工具名匹配 "Bash" / "Write|Edit"
  rule_type VARCHAR(20),                -- builtin_policy / custom_script
  builtin_policy_key VARCHAR(60) NULL,  -- 内置策略标识
  params JSON,                          -- 策略参数(黑名单正则/路径前缀...)
  script TEXT NULL,                     -- 仅 custom_script; 沙箱执行
  enabled BOOLEAN DEFAULT 1,
  locked BOOLEAN DEFAULT 0,             -- 管理员强制启用, 用户不可关(如 audit_log)
  created_at, updated_at
)

permission_rules(
  id PK, owner_type, owner_id,
  tool_pattern VARCHAR(200),            -- "Bash(git push:*)" / "Write" / "*"
  decision VARCHAR(10),                 -- allow / deny / ask
  priority INT DEFAULT 0,
  source VARCHAR(20) DEFAULT 'manual',  -- manual / remembered(审批时"记住")
  created_at
)

permission_settings(
  owner_type, owner_id PK组合,
  mode VARCHAR(30) DEFAULT 'default',   -- default/acceptEdits/plan/bypassPermissions
  max_turns INT, run_timeout_sec INT,
  updated_at
)

user_permission_profiles(               -- 管理员为每个用户设定的"权限天花板"
  user_id VARCHAR(36) PK,               -- 合并优先级最高, 用户自身配置不可越过
  mode_ceiling VARCHAR(30) DEFAULT 'default',  -- 用户可选权限模式的上限
  tool_grants JSON,                     -- {"bash":false,"file_write":true,"web_search":true,
                                        --  "workspace_view":true(工作区文件树可见性),...}
  daily_token_limit BIGINT NULL,        -- NULL=不限
  daily_run_limit INT NULL,
  show_tokens BOOLEAN DEFAULT 1,        -- 是否向该用户展示 token 用量
  show_cost BOOLEAN DEFAULT 0,          -- 是否展示费用(内部模型阶段默认关闭)
  note VARCHAR(300),                    -- 管理员备注
  updated_by VARCHAR(36), updated_at
)

app_settings(                           -- 全局默认(单行表或 key-value)
  key VARCHAR(60) PK, value JSON, updated_by, updated_at
  -- 预置: web_search_default(bool)     未配置 profile 的用户是否可用联网搜索
  --      show_tokens_default(bool)    默认是否展示 tokens
  --      show_cost_default(bool)      默认是否展示费用(当前=false, 内部模型)
  --      default_mode_ceiling(str)
)

pending_approvals(
  id PK, run_id FK, thread_id FK,
  tool_name VARCHAR(100), tool_input JSON,
  status VARCHAR(20),                   -- pending/approved/denied/timeout
  decided_by VARCHAR(36) NULL, remember BOOLEAN DEFAULT 0,
  created_at, decided_at
)

audit_logs(                             -- 只增不改
  id PK, user_id, thread_id, run_id,
  event VARCHAR(40),                    -- tool_call/tool_denied/approval/login/config_change
  detail JSON, created_at,
  INDEX ix_audit_user_time (user_id, created_at)
)
```

## 3. SQLite → MySQL 迁移方案

1. **结构**：同一套 Alembic 迁移脚本直接在 MySQL 上 `alembic upgrade head`（这是坚持第 1 节规范的回报）。
2. **数据**：基于 SQLAlchemy Core 写通用导出/导入脚本，按外键拓扑顺序
   `users → projects → threads → branches → messages → runs → 其余` 逐表搬运；
   MySQL 导入期间 `SET FOREIGN_KEY_CHECKS=0`，完成后恢复并全表行数 + 抽样校验和比对。
3. **重写点（预期内）**：全文检索实现（fts5 → FULLTEXT/Meilisearch）；
   高并发下把进程内锁/Future 换 Redis。
4. **演练**：切换前在预发环境做一次全量迁移 + 回放最近 N 天流量验证。
